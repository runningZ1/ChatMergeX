## 1. 产品概述

### 1.1 产品定位

ChatMergeX 是一个**多平台 AI 对话记录统一管理平台**，帮助用户将分散在不同 AI 平台的对话记录集中管理，并以知识管理的方式进行组织和检索。

### 1.2 核心价值主张

- **统一入口**：一个平台管理所有 AI 对话记录
- **自动同步**：浏览器插件实时自动同步，无需手动操作
- **知识管理**：用文件夹和搜索的方式管理对话，像管理笔记一样管理 AI 对话
- **无缝衔接**：点击链接直接在内置窗口继续对话，无需切换标签页
- **隐私优先**：所有数据本地存储，不上传云端

### 1.3 目标用户

**主要用户群体**：个人用户
- AI 工具重度使用者
- 开发者、研究人员、内容创作者
- 需要频繁在多个 AI 平台间切换的用户
- 希望系统化管理 AI 对话历史的用户

### 1.4 产品愿景

成为 AI 时代的"对话记录管理中心"，让用户的 AI 对话历史成为可检索、可复用的知识资产。

## 2. 市场分析

### 2.1 用户痛点

1. **对话记录分散**
   - 用户在多个 AI 平台（ChatGPT、豆包、Gemini 等）都有对话记录
   - 需要记住每个对话在哪个平台，查找困难

2. **缺乏有效管理**
   - 各平台自带的对话管理功能简陋
   - 无法按主题、项目分类整理
   - 历史对话难以检索

3. **切换成本高**
   - 需要打开多个标签页或窗口
   - 频繁切换影响工作效率

4. **数据不可控**
   - 对话记录存储在各平台服务器
   - 担心数据丢失或隐私泄露

### 2.2 竞品分析

**参考产品**：Twitter 书签同步工具

**相似之处**：
- 三栏布局设计（导航 - 列表 - 详情）
- 卡片式内容展示
- 文件夹/分类管理
- 搜索和筛选功能
- 自动同步机制

**差异化**：
- **多平台支持**：不是单一平台，而是聚合多个 AI 平台
- **富文本内容**：需要保存代码块、图片等复杂格式
- **内置对话窗口**：可以直接在应用内继续对话
- **本地优先**：数据完全本地存储，无需注册账号

### 2.3 市场机会

- AI 工具使用量爆发式增长
- 用户对 AI 对话管理的需求尚未被满足
- 现有工具多为单平台，缺乏跨平台整合方案

## 3. 功能需求

### 3.1 核心功能（MVP 版本）

#### 3.1.1 多平台对话同步

**功能描述**：自动同步用户在各 AI 平台的对话记录到本地

**MVP 支持平台**：
- ✅ ChatGPT（优先）
- ✅ 豆包（优先）
- ⏳ 元宝（后续）
- ⏳ Gemini（后续）
- ⏳ Grok（后续）

**技术实现**：
- 通过浏览器插件读取用户已登录平台的 Cookie/Session
- 监听页面变化，实时捕获新对话
- 将对话内容解析并存储到本地 IndexedDB

**同步机制**：
- **触发方式**：实时自动同步
- **同步内容**：
  - 对话标题
  - 完整对话内容（问题 + 回答）
  - 富文本格式（文本、代码块、图片、链接等）
  - 时间戳
  - 原平台链接

**用户流程**：
1. 用户安装浏览器插件
2. 插件检测用户已登录的 AI 平台
3. 用户授权插件读取对话数据
4. 插件自动开始同步历史对话
5. 后续对话实时同步到 Web 应用

#### 3.1.2 知识管理系统

**功能描述**：用文件夹的方式组织和管理对话记录

**核心功能**：

1. **文件夹管理**
   - 创建/重命名/删除文件夹
   - 支持多级文件夹（树形结构）
   - 拖拽对话到文件夹
   - 文件夹折叠/展开

2. **对话列表视图**
   - 表格形式展示对话列表
   - 列信息：
     - 平台图标
     - 对话标题
     - 消息数量
     - 创建时间
     - 最后更新时间
     - 所属文件夹
   - 支持排序（按时间、标题、平台）
   - 支持多选

3. **基础操作**
   - **查看**：点击对话查看完整内容
   - **编辑**：修改对话标题
   - **移动**：移动到其他文件夹
   - **删除**：删除对话记录
   - **导出**：导出为 Markdown/JSON/HTML

4. **批量操作**
   - 批量移动到文件夹
   - 批量删除
   - 批量导出

#### 3.1.3 搜索与筛选

**功能描述**：快速定位历史对话

**搜索功能**：
- 全文搜索（标题 + 内容）
- 实时搜索结果高亮
- 搜索历史记录

**筛选功能**：
- 按平台筛选（ChatGPT、豆包等）
- 按时间范围筛选（今天、本周、本月、自定义）
- 按文件夹筛选
- 组合筛选

**排序功能**：
- 按创建时间（升序/降序）
- 按更新时间（升序/降序）
- 按标题（A-Z）

#### 3.1.4 对话详情展示

**功能描述**：完整展示对话内容，保留原始格式

**展示内容**：
- 对话元信息（平台、时间、链接）
- 完整对话历史（问答对）
- 富文本渲染：
  - Markdown 格式
  - 代码高亮（支持多种语言）
  - 图片展示
  - 链接可点击

**交互功能**：
- 复制单条消息
- 复制整个对话
- 导出对话
- 在原平台打开

#### 3.1.5 内置对话窗口

**功能描述**：点击链接在应用内打开对话，无需切换标签页

**技术实现**：
- 使用 iframe 嵌入原平台网页
- 悬浮窗口形式展示
- 可拖拽、调整大小
- 可最小化/最大化/关闭

**用户流程**：
1. 用户在对话列表点击"继续对话"按钮
2. 应用内弹出悬浮窗口
3. 窗口中加载原平台对话页面
4. 用户可以直接在窗口内继续对话
5. 新对话自动同步回应用

**备选方案**：
- 如果 iframe 被 CSP 策略阻止，降级为新标签页打开

### 3.2 后续功能（未来版本）

#### 3.2.1 标签系统
- 为对话添加多个标签
- 按标签筛选和分组

#### 3.2.2 智能分类

- 自动建议分类和标签
- 相似对话推荐

#### 3.2.3 数据统计
- 对话数量统计
- 各平台使用频率
- 时间分布图表
- 主题词云

#### 3.2.4 高级搜索
- 语义搜索（AI 理解搜索意图）
- 代码搜索（专门搜索代码片段）
- 正则表达式搜索

#### 3.2.5 协作功能
- 分享对话链接
- 导出为公开页面
- 团队共享文件夹

## 4. 技术架构

### 4.1 技术栈选型

#### 4.1.1 前端技术栈

**核心框架**：
- **React 18** + **TypeScript** + **Vite**
  - 理由：生态成熟，shadcn/ui 原生支持，开发体验好

**UI 组件库**：
- **shadcn/ui**（基于 Radix UI + Tailwind CSS）
  - 理由：现代化设计，可定制性强，无运行时依赖

**状态管理**：
- **Zustand** 或 **Jotai**
  - 理由：轻量级，API 简洁，适合中小型应用

**路由**：
- **React Router v6**
  - 理由：标准路由方案，功能完善

**数据存储**：
- **IndexedDB**（使用 **Dexie.js** 封装）
  - 理由：浏览器原生支持，容量大（几百 MB），支持复杂查询

**其他工具库**：
- **date-fns**：日期处理
- **react-markdown**：Markdown 渲染
- **prism-react-renderer**：代码高亮
- **react-window**：虚拟滚动（优化大列表性能）

#### 4.1.2 浏览器插件技术栈

**Manifest 版本**：V3（Chrome 最新标准）

**目标浏览器**：
- Chrome 90+（优先）
- Edge 90+（后续）

**核心技术**：
- **Content Script**：注入到 AI 平台页面，捕获对话内容
- **Background Service Worker**：后台处理同步逻辑
- **chrome.storage API**：存储插件配置
- **chrome.runtime.sendMessage**：与 Web 应用通信

**跨域通信**：
- 插件 → Web 应用：通过 `window.postMessage`
- Web 应用 → 插件：通过 `chrome.runtime.sendMessage`

### 4.2 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户浏览器                              │
│                                                           │
│  ┌─────────────────┐         ┌─────────────────┐        │
│  │  AI 平台页面     │         │  ChatMergeX      │        │
│  │  (ChatGPT/豆包)  │         │  Web 应用        │        │
│  │                 │         │                 │        │
│  │  ┌───────────┐  │         │  ┌───────────┐  │        │
│  │  │ Content   │  │         │  │  React    │  │        │
│  │  │ Script    │◄─┼─────────┼─►│  App      │  │        │
│  │  └───────────┘  │         │  └───────────┘  │        │
│  └─────────────────┘         │        │        │        │
│           │                  │        ▼        │        │
│           │                  │  ┌───────────┐  │        │
│           │                  │  │ IndexedDB │  │        │
│           │                  │  │ (Dexie.js)│  │        │
│           │                  │  └───────────┘  │        │
│           │                  └─────────────────┘        │
│           ▼                                              │
│  ┌─────────────────┐                                    │
│  │  Background     │                                    │
│  │  Service Worker │                                    │
│  └─────────────────┘                                    │
└─────────────────────────────────────────────────────────┘
```

**数据流**：
1. 用户在 AI 平台对话
2. Content Script 监听页面变化，捕获对话内容
3. 发送到 Background Service Worker 处理
4. 通过 postMessage 发送到 Web 应用
5. Web 应用解析并存储到 IndexedDB
6. UI 实时更新显示新对话

### 4.3 部署架构

**产品形态**：纯 Web 应用（在线访问）+ 浏览器插件

**部署方案**：
- **Web 应用**：部署到静态托管服务（Vercel/Netlify/GitHub Pages）
- **浏览器插件**：发布到 Chrome Web Store

**数据存储**：
- 完全本地存储（浏览器 IndexedDB）
- 无需后端服务器
- 无需用户注册登录

## 6. 用户流程

### 6.1 首次使用流程

1. **访问 Web 应用**
   - 用户通过浏览器访问 ChatMergeX 网站
   - 看到欢迎页面和产品介绍

2. **安装浏览器插件**
   - 引导用户点击"安装插件"按钮
   - 跳转到 Chrome Web Store
   - 用户安装插件

3. **插件配置**
   - 插件自动检测用户已登录的 AI 平台
   - 显示配置页面，列出检测到的平台
   - 用户选择要同步的平台

4. **开始同步**
   - 插件开始同步历史对话
   - 显示同步进度（已同步 X/Y 条对话）
   - 同步完成后跳转到 Web 应用

5. **开始使用**
   - 用户看到同步好的对话列表
   - 可以开始浏览、搜索、管理对话

### 6.2 日常使用流程

1. **自动同步**
   - 用户在 AI 平台正常对话
   - 插件后台自动同步到本地
   - 无需手动操作

2. **查看对话**
   - 用户打开 ChatMergeX Web 应用
   - 浏览对话列表
   - 使用搜索/筛选快速定位

3. **管理对话**
   - 创建文件夹分类
   - 拖拽对话到文件夹
   - 编辑对话标题
   - 导出重要对话

4. **继续对话**
   - 点击"继续对话"按钮
   - 在内置窗口继续对话
   - 新对话自动同步回应用

### 6.3 核心使用场景

#### 场景 1：查找历史对话
> 用户记得几天前和 ChatGPT 讨论过一个技术问题，想找回那个对话。

**流程**：
1. 打开 ChatMergeX
2. 在搜索框输入关键词（如"React Hooks"）
3. 筛选平台为 ChatGPT
4. 筛选时间范围为"最近一周"
5. 在结果中找到对话，点击查看详情
6. 复制需要的代码片段

#### 场景 2：整理项目对话
> 用户正在做一个新项目，想把相关的 AI 对话都整理到一起。

**流程**：
1. 创建新文件夹"项目 A"
2. 使用搜索找到相关对话
3. 批量选择多个对话
4. 拖拽到"项目 A"文件夹
5. 后续对话手动移动到该文件夹

#### 场景 3：跨平台对比
> 用户想对比 ChatGPT 和豆包对同一问题的回答。

**流程**：
1. 在 ChatGPT 问问题，插件自动同步

3. 打开 ChatMergeX，搜索关键词
4. 并排查看两个平台的回答
5. 复制最满意的答案

## 7. 非功能需求

### 7.1 性能要求

- **加载速度**：首屏加载时间 < 2 秒
- **列表渲染**：支持 10,000+ 对话流畅滚动（使用虚拟滚动）
- **搜索响应**：搜索结果返回时间 < 500ms
- **同步速度**：单条对话同步延迟 < 5 秒

### 7.2 安全性要求

- **数据存储**：所有数据本地存储，不上传服务器
- **Cookie 安全**：Cookie 仅用于同步，不外传
- **XSS 防护**：对用户输入和对话内容进行转义
- **CSP 策略**：实施内容安全策略防止注入攻击

### 7.3 隐私保护

- **无需注册**：不收集用户个人信息
- **本地优先**：数据完全存储在用户设备
- **透明性**：明确说明数据存储位置和使用方式
- **用户控制**：用户可随时导出或删除所有数据

### 7.4 兼容性

- **浏览器**：Chrome 90+, Edge 90+
- **操作系统**：Windows 10+, macOS 10.15+, Linux
- **屏幕分辨率**：最小支持 1280x720

### 7.5 可用性

- **国际化**：支持中文、英文双语
- **无障碍**：支持键盘导航和屏幕阅读器
- **错误处理**：友好的错误提示和恢复机制
- **离线支持**：Web 应用支持离线访问（PWA）

### 7.6 可维护性

- **代码规范**：使用 ESLint + Prettier 统一代码风格
- **类型检查**：TypeScript 严格模式
- **测试覆盖**：单元测试覆盖率 > 70%
- **文档完善**：代码注释和开发文档

## 9. 风险和挑战

### 9.1 技术风险

#### 风险 1: 平台反爬虫机制
**描述**：AI 平台可能有反爬虫策略，阻止插件读取数据

**影响**：无法实现自动同步

**应对方案**：
- 优先使用官方 API（如果可用）
- 使用更隐蔽的数据抓取方式
- 降级到手动导入方案
- 与平台沟通获取官方支持

**优先级**：高

#### 风险 2: Cookie/Session 读取限制
**描述**：浏览器安全策略可能限制插件读取敏感信息

**影响**：无法实现自动登录和同步

**应对方案**：
- 使用官方 OAuth 授权流程
- 让用户手动提供 API Token
- 使用 Content Script 在页面上下文中读取

**优先级**：高

#### 风险 3: iframe CSP 策略阻止
**描述**：AI 平台的 CSP 策略可能阻止 iframe 嵌入

**影响**：内置对话窗口无法使用

**应对方案**：
- 检测 CSP 错误，自动降级
- 提供新标签页打开的备选方案
- 提示用户安装浏览器扩展来绕过限制

**优先级**：中

#### 风险 4: 页面结构频繁变化
**描述**：AI 平台更新页面结构，导致抓取失败

**影响**：同步功能失效

**应对方案**：
- 实现灵活的内容解析器
- 建立版本兼容机制
- 快速响应和更新插件
- 社区众包维护

**优先级**：中

### 9.2 产品风险

#### 风险 5: 用户隐私担忧
**描述**：用户担心插件读取敏感对话内容

**影响**：用户不愿意安装插件

**应对方案**：
- 开源代码，接受社区审计
- 明确说明数据本地存储
- 提供详细的隐私政策
- 支持可选的端到端加密

**优先级**：高

#### 风险 6: 用户需求验证不足
**描述**：产品功能不符合用户实际需求

**影响**：产品使用率低

**应对方案**：
- 快速发布 MVP 验证需求
- 收集用户反馈持续迭代
- 建立用户社区
- A/B 测试新功能

**优先级**：中

### 9.3 市场风险

#### 风险 7: AI 平台推出官方类似功能
**描述**：AI 平台自己推出跨平台管理工具

**影响**：产品失去竞争力

**应对方案**：
- 专注差异化功能（本地存储、隐私保护）
- 建立先发优势和用户基础
- 快速迭代，保持领先
- 提供平台无法提供的高级功能

**优先级**：低

#### 风险 8: 竞品出现
**描述**：其他团队开发类似产品

**影响**：市场竞争加剧

**应对方案**：
- 保持产品创新和体验优势
- 建立品牌和社区
- 开源部分代码吸引贡献者
- 快速响应市场变化

**优先级**：中

### 9.4 法律风险

#### 风险 9: 违反平台服务条款
**描述**：自动抓取数据可能违反 AI 平台的 ToS

**影响**：法律纠纷或被平台封禁

**应对方案**：
- 仔细研究各平台的 ToS
- 咨询法律专家
- 仅抓取用户自己的数据
- 与平台沟通获取许可

**优先级**：高
